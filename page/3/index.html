  1. 执行 new Widget
  2. 构造 shared_ptr<Widget>
  3. 调用 priority()

因为C++不同于其他语言，函数参数的计算顺序很大程度上决定于编译器，编译器认为顺序应当是1, 3, 2，即：

  1. 执行 new Widget
  2. 调用 priority()
  3. 构造 shared_ptr<Widget>

那么如果 priority抛出了异常，新的 Widget 便永远地找不回来了。虽然我们使用了智能指针，但资源还是泄漏了！
